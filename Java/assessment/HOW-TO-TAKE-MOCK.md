## â±ï¸ Time Management Tips

### Part 2: Rapid-Fire Round Strategy
- **Don't overthink** - Answer quickly (30 seconds per question max)
- **Skip if unsure** - Better to mark "Don't know" than guess
- **This tests instant recall** - Like a real interview

### Part 6: DSA Problems Strategy
- **Set timer** - 6-8 minutes per problem
- **If stuck >5 min** - Move to next, come back later
- **Write algorithm first** - Then code
- **Time complexity matters** - Always mention it

## ğŸ“ Answering Guidelines

### For Theory Questions:
- **Be specific** - Don't just say "it's used for X", explain HOW
- **Use examples** - Code snippets help
- **Show depth** - Explain internals if you know them

### For Code Analysis:
- **Trace execution** - Step by step
- **Explain "why"** - Not just "what"
- **Predict output** - Then verify mentally

### For DSA Problems:
- **Think out loud** (write notes)
- **Consider edge cases** - Empty array? Null? Single element?
- **Optimize** - Can you do better than brute force?
